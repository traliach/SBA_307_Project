# Consulting / Engagements Positioning Note

Achille is open to consulting work, but the site should not publish fixed prices
or package ranges right now. Pricing depends on the actual scope, timeline,
risk, environment, and expected deliverables.

Current positioning:

- Keep consulting visible as an option across Services, Contact, and CTA copy.
- Use "Request scope", "Request a call", or "Discuss the work" instead of "Buy",
  "Start at $X", or package pricing.
- Avoid fixed-price cards unless Achille later confirms a repeatable offer.
- Do not add `/pricing` to the nav.
- Do not make price the focus of the consulting message.

If an Engagements page is added later, it should be scope-first, not price-first:

- Platform reliability consultation
- CI/CD and deployment workflow review
- AWS / Terraform infrastructure support
- Kubernetes and observability support
- Full-stack delivery with DevOps handoff

Each engagement card should explain:

- Best for
- Typical problems
- Possible deliverables
- Discovery questions
- CTA: "Request scope"

Pricing copy should stay custom:

> Pricing depends on scope. Share the current system, target outcome, and
> timeline, and I will suggest the shortest practical path.

No backend, database, auth, CI, or infrastructure changes are required for this.
